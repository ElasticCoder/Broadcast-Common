Bandwidth Tuner

You can use the RealFlash bandwidth tuner to adjust the streaming bit rate for a Shockwave Flash file and affect the RealFlash presentation preroll. Bit rate and preroll are inversely related. The higher the bit rate, the lower the initial buffering. The tuner lets you measure these properties and adjust them for an optimal balance.

Opening a Shockwave Flash File

Do the following to open a Shockwave Flash file in the RealFlash bandwidth tuner:

  1. Double-click the tuner icon to display the utility's main window.

  2. Select the Shockwave Flash (.swf) file by entering its path and name in the Input File field. You can also browse to the file by choosing File>Open or clicking the Open button by the Input File field.

  3. Choose Options>Target Bit Rate to set the combined target bit rate for the Shockwave Flash file and the RealAudio soundtrack. The default target is 20 Kbps.

Reading the Tuner Graph

The RealFlash tuner graph's horizontal scale represents the presentation time in seconds. Its vertical scale depicts streaming bit rate. This bit rate scale is based on the target bit rate you chose. If you chose a 20 Kbps target rate, for example, the graph's vertical scale extends to 15, because at least 5 Kbps must be reserved for the RealAudio soundtrack.

On the graph's left side, a red region depicts the presentation's initial buffering time. Text below this red region tells how many seconds of buffering the file requires. The Buffer Time field in the File Attributes region also lists this time. You cannot change the buffer time directly. You adjust only the bit rate, which modifies the buffer time automatically.

Adjusting the File's Bit Rate

You can change the Shockwave Flash file's streaming bit rate with the spinner controls in the File Attributes region. You can also adjust it by moving the bit rate slider at the left of the graph:

  • Increasing the bit rate by moving the slider up decreases the buffer time.

  • Deceasing the bit rate by moving the slider down increases the buffer time.

  • The buffer time should be under 10 seconds.

The graph depicts the changes you make, and the modified bit rate and buffer time display in the File Attributes region.

Using the RealAudio Codec Recommendation

As you adjust the file bit rate, a text field below the graph recommends which RealAudio codec to use for the presentation soundtrack. The utility determines this by subtracting the bit rate of the Shockwave Flash file from the target bit rate you set with Options>Target Bit Rate. It then displays the highest bit rate codec that does not exceed the remaining available bit rate.

Tip
To maximize presentation efficiency, set the file bit rate as high as possible for your chosen codec. If you plan to use a 12 Kbps codec, for example, tune the file until that codec recommendation displays. Then move the bit rate slider as high as possible without changing the recommended codec.

Showing Detail

Click Options>Show Detail to display the file's second-by-second streaming bandwidth usage as shown in the next figure. The line that appears within the graph typically shows spikes that indicate high bandwidth usage. You can click on this line at any point to find out what frame and time in the presentation timeline the spikes occur.

Bandwidth Consumption Detail

Keep in mind that the spikes do not directly correspond to the frame sizes. You might see a spike at 5 seconds in the timeline, for example, when the frame size is small. Yet there may be no spike at 12 seconds when the frame size is large. This is because RealServer and RealPlayer preload data to even out bit rate consumption over the presentation timeline. The spike at 5 seconds may represent data being downloaded in advance for a scene change that occurs at the 12-second mark.

Because RealSystem preloads data, you typically see the Shockwave Flash bit rate consumption drop off to near zero by the end of the presentation. This simply means that by this point RealServer has downloaded all graphical scene data and needs to stream only the small amount of control information required to manipulate the scene data for each frame.

Saving the Tuned File

When you reach the optimal balance between bit rate and buffering, save the output by typing a new name in the Output File field and clicking Apply. This saves the new file to the directory that contains the input file. To overwrite the input file, save the output file to the same file name.

You can also browse to a directory by choosing File>Save or clicking the Save As button by the Output File field. In this case, choose the directory and enter the file name in the browse dialog box. The click Apply on the main window.

Adjusted Output File
