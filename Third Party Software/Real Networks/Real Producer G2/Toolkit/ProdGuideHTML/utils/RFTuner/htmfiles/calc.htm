Bit Rate Calculation Spreadsheet

The RealFlash bit rate spreadsheet indicates bandwidth spikes and shows how changing certain Flash attributes affects playback quality. This helps you determine the trade-off between bit rate and buffering. Use it to get an in-depth look at RealFlash playback. With this information, you can modify your presentation by changing the Flash source file. Using the spreadsheet is not necessary, however. You can simply use the RealFlash tuner to evaluate and adjust the presentation preroll.

Importing RealFlash Data

To import presentation data into the spreadsheet:

  1. Create a movie report when exporting your Shockwave Flash file.

  2. In Microsoft Word, open the movie report, which is a text file in the directory where you exported the Shockwave Flash file.

  3. Holding down Alt (Windows) or Option (Macintosh), click and drag to select the Frame Bytes column.

  4. Choose Edit>Copy.

  5. In Microsoft Excel, open flashspreadsheet.xlt.

  6. Place the cursor in the first cell of the Frame Bytes column as shown in the following figure and select Edit>Paste.
    Pasting Frame Byte Information

  7. Choose NonBlanks from the pull-down menu at the top of the Frame Bytes column as shown in the next figure. This makes visible only the spreadsheet rows that have values.
    Choosing Nonblank Information

  8. Scroll down to determine the number of frames in the animation (highlighted in blue). Enter this value in the Total Number of Frames field.
    Total Number of Frames

  9. Check that the FPS field has the appropriate frames per second rate, typically 7.

  10. In the RealAudio Bit Rate field, enter the Kbps rating for the RealAudio codec you plan to use to encode the presentation soundtrack.

After you have entered the initial spreadsheet data, you can analyze it and change the fields displayed in green to determine how to optimize your Flash file. The goal is to turn all values in the Bandwidth Available column to black. Red values indicate where buffering will occur.

Spreadsheet Fields

The following sections explain the spreadsheet fields in detail. Each section tells whether you can edit the field and explains what type of information the field contains.

Frame # (not editable)

Numbers in these fields correspond to the frame numbers in the Flash timeline.

Second (not editable)

Numbers in these fields tell at which second into the RealFlash presentation each frame appears.

Frame Bytes (editable)

These fields indicate the number of bytes consumed by each frame of the animation. You can change values manually to see how bandwidth consumption changes.

Note
Because Flash expresses frame size in bytes (1 byte equals 8 bits), the spreadsheet uses bytes as the frame size value, automatically converting between bytes and Kilobits in the various fields as necessary.

Bandwidth Available (not editable)

These fields tell how much extra bandwidth in Kilobits is available during playback of the frame. If the number is black, the frame does not exceed the available bandwidth. If the number is red, the frame exceeds the available bandwidth and buffering will occur.

You can modify these values by adjusting the Buffer Time and Max Bandwidth fields. If certain frames consume a large amount of bandwidth, check the original movie report to determine what elements are downloaded in that frame. You may be able to edit the Flash source file to eliminate or modify those elements.

FPS (editable)

This field corresponds to the frames per second rate set in your Flash animation.

Overhead (not editable)

This field is a percentage of available bandwidth included to compensate for network packet data used in routing the presentation from RealServer to RealPlayer.

SWF Bit Rate (not editable)

This field gives the average bit rate of the Flash animation. It is calculated by dividing the file size in Kilobits by the number of seconds the presentation lasts.

Total Bit Rate (not editable)

This field indicates the Kilobit rate at which RealServer will stream the RealFlash presentation. This is a combined total for the Shockwave Flash and RealAudio streams.

Total Number of Frames (editable)

This field gives the total number of frames in the Flash file.

Buffer Time (editable)

This field indicates the number of seconds RealPlayer buffers data before beginning playback of the presentation. This buffering, also known as "preroll," is required for any presentation and should be under 10 seconds for optimal presentation performance. It should not exceed 15 seconds. You can adjust the preroll by changing the file bandwidth with the RealFlash bandwidth tuner.

Audio Bit Rate (editable)

This is the bit rate of the RealAudio file in Kilobits. The codec you use when encoding the file in the RealAudio format determines this rate. You can edit this field to see how encoding the audio with a different codec affects the bandwidth available for the animation stream.

Max Bandwidth for the SWF File (editable)

In this field, enter the maximum bandwidth in Kilobits you want to allocate to the animation stream. You can adjust this bandwidth in the file with the RealFlash bandwidth tuner.

RealFlash Bytes Consumed (not editable)

This graphic is available through a tab at the bottom of the spreadsheet. It is provided for informational use only and graphs the number of bytes each frame requires. These values correspond to the Frame Bytes column.

RealFlash Available Bandwidth (not editable)

This graphic is available through a tab at the bottom of the spreadsheet. For informational use only, it shows the Kilobytes of extra bandwidth available in each frame. These values correspond to the Bandwidth Available column.
